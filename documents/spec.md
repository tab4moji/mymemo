# ソフトウェア仕様書 (spec.md)

⚠️ このドキュメントはリバースエンジニアリングにより自動生成された。

## 目的
ローカルまたはリモートの Markdown ファイルを Fetch し、シンタックスハイライト、コードコピー、コード折りたたみなどの高機能なビューワー機能を備えた HTML 表示を提供する。

## 機能要件
- **FR-001: 対象ファイルの動的決定**
  URL パラメータ `?content=<filename>` を参照し、読み込む Markdown ファイルを決定する（未指定時は `README` がデフォルト）。
- **FR-002: Markdown ファイルの Fetch**
  指定されたファイルを `${filename}.md` から Fetch し、キャッシュを無効化するためのタイムスタンプパラメータ (`?t=<epoch>`) を付与する。
- **FR-003: ネストされたコードブロックの修復**
  Fetch した生の Markdown テキストに対し、多重にネストされたコードブロックのバッククォートの長さを自動調整し、パース時の崩れを防ぐ (`fixNestedCodeBlocks`)。
- **FR-004: Markdown のパースおよびカスタムコードレンダリング**
  `marked.js` を用いて HTML へ変換する。その際、コードブロック記法 ` ```lang:title ` に対応し、言語とタイトルを抽出する。
- **FR-005: サニタイズ処理**
  パース後の HTML に対し、`DOMPurify` を使用してクロスサイトスクリプティング (XSS) 脆弱性を排除する。
- **FR-006: 外部リンクの安全対策**
  すべての `<a>` タグに対し、`target="_blank"` および `rel="noopener noreferrer"` を動的に追加する。
- **FR-007: コードブロックの折りたたみ (Snip) 機能**
  コードブロックの総行数が閾値 (初期値: 20行) を超える場合、冒頭と末尾の一定行 (初期値: 10行) のみを表示し、中間を折りたたむ。クリックで全展開できるようにする。
- **FR-008: コードコピー機能**
  各コードブロックに「Copy」ボタンを設置し、クリック時にクリップボードへコードの全テキストをコピーする。
- **FR-009: ドキュメントタイトルの動的変更**
  Markdown の読み込み成功時に、ブラウザの `<title>` をファイル名に書き換える。
- **FR-010: エラーハンドリング**
  ファイルの取得失敗やスクリプトエラー発生時に、画面にエラーメッセージを表示する。
- **FR-011: カラーテーマ切り替え対応**
  ダークテーマとライトテーマの分離されたスタイルシートを提供し、必要に応じて切り替えができるようにする。
- **FR-012: 見出しアンカーとURLショートカットコピー機能**
  HTML パース時に各見出し（H1〜H6）に対して一意なアンカー ID を生成・付与し、見出しの横にリンク用のタグ（アイコン等）を配置する。タグホバー時にコピーカーソルを表示し、クリック時にそのアンカー付きの完全なページ URL をクリップボードにコピーする。
  アンカー ID は、見出しの正規化文字列（小文字化、空白のハイフン化、記号除去）から生成された URL-safe な Base64 で 8 文字以内のハッシュ値とする。また、ドキュメント内の見出しリンク（ページ内アンカーおよび他ドキュメントへのアンカー）についても、遷移が機能するように対応するハッシュ値に変換する。

## 非機能要件
- **NFR-001: 外部 CDN 依存管理**
  `marked.js`, `DOMPurify`, `highlight.js` の外部ライブラリを適切に読み込み、競合や未定義エラーを防ぐ。
- **NFR-002: レスポンシブデザイン**
  モバイル端末およびPC画面の双方で最適な余白とフォントサイズで表示されること（CSS メディアクエリ対応）。
- **NFR-003: パフォーマンス**
  追加のクライアントサイド処理による描画遅延を最小限に抑える。
