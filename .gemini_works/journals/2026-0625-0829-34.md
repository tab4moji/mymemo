## 2. 試行錯誤と決定事項
* 単一ファイルで構成されていた高機能な `markdown.html` を、保守性向上のため `markdown.html` (骨組み)、`markdown.css` (共通レイアウト)、`markdown_dark.css` (ダークテーマ用)、`markdown_light.css` (ライトテーマ用)、`markdown_parse.js` (Markdown解析・コードブロック修復)、`markdown.js` (UI動作制御) に分割した。
* CSSのインポート順序について、`markdown.css` の中で `@import` を用いて GitHub 互換の外部 CSS をインポートし、テーマファイル（`markdown_dark.css` や `markdown_light.css`）の中で `highlight.js` のテーマ CSS をインポートする構成をとることで、各ファイルに閉じたテーマ設定を実現した。
* ダークテーマとライトテーマの動的切り替え機能をサポートするため、UIにフローティング切り替えボタン（FAB）を設置し、`markdown.js` にて `localStorage` や OS 設定 (`prefers-color-scheme`) に基づき動的に CSS ファイルを書き換える制御を組み込んだ。
* 開発ライフサイクルルールに基づき、`documents/` 配下に `spec.md`, `modules.md` 等のドキュメントをリバースエンジニアリングで自動生成した。
