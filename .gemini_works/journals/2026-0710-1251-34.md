## 2. 試行錯誤と決定事項
* **ハッシュ化アルゴリズムの選定と実装:**
  クライアントサイドの JavaScript (同期処理) で高速かつ衝突が極めて少ないハッシュ値を生成するため、FNV-1a 64bit ハッシュアルゴリズムを JavaScript で実装した。64bit の出力のうち上位 6バイト (48ビット) を抽出し、URL-safe Base64 にエンコードすることで、文字数がちょうど 8文字になるよう調整した。
* **見出し ID の変更:**
  `markdown_parse.js` の見出しカスタムレンダリング (`renderer.heading`) にて、見出しの正規化文字列（小文字化、空白のハイフン化、特定記号の除去）からハッシュ値を生成し、ID属性およびコピー用のアンカー属性に付与するようにした。
* **リンクの書き換え処理の追記:**
  見出しIDをハッシュ化すると、Markdown内の `[テキスト](#見出し)` などのリンクが機能しなくなる。そのため `markdown.js` のリンク変換部にて、ページ内アンカーや他ドキュメントへのアンカー付き相対リンクを見つけて、そのアンカー文字列に対しても同一のハッシュ化処理を行い、変換後のハッシュ化されたIDへと自動置換するようにした。これで遷移とスクロールが完全に動作する状態を維持した。
