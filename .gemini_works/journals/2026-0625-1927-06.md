# 2026-06-25-1927-06: MarkdownViewerにおける[text](url)リンク機能の不具合修正

## 2. 試行錯誤と決定事項
* **原因の調査**:
  * 既存のコードでは、すべての `a` タグに対して一律で `target="_blank"` および `rel="noopener noreferrer"` が設定されていた。
  * このため、同一ファイル内へのアンカーリンク（`#something`）に対しても新しいタブで開く挙動になっており、スクロール遷移が機能しなくなっていた。
  * また、 `.md` や `.markdown` で終わるローカルの相対ファイルリンク（例: `aipc.md`, `../README.md`）をクリックした際、ビューア（`markdown.html?content=xxx`）を介さずに生ファイルに直接ジャンプしようとしてしまうため、ブラウザ側で生テキストが表示されるかエラーになり、正常に閲覧できなかった。
* **解決策の策定**:
  * `markdown.js` の `loadMarkdown` 内で `a` タグの属性設定をカスタマイズ。
  * **アンカーリンク (`#` で始まる)**: `target` 属性などを設定せず、そのままブラウザ標準のスクロール挙動に任せる。
  * **外部リンク (`http://`, `https://` で始まる)**: 従来通り安全に別タブで開く（`target="_blank"`, `rel="noopener noreferrer"`）。
  * **ローカルマークダウンファイル (`.md` や `.markdown` で終わる)**: 現在の表示ファイル名（`filename`）からの相対パスを `URL` API を用いて正しく解決し、 `markdown.html?content=解決したパス（拡張子なし）` の形式に `href` を書き換え、同じタブ（または `target` なし）で開くようにした。
  * **その他のローカルファイル (PDF等)**: 安全に別タブで開く。
* **決定事項**:
  * `URL` API (`new URL(baseDir + pathPart, 'http://dummy/')`) を用いることで、ディレクトリを遡る相対パス（`../` 等）やハッシュ付きの相対パス（`aipc.md#setup`）も簡潔に解決可能とした。

## 3. エラー調査結果
- 原因: `a` タグの一律 `target="_blank"` 適用、およびローカル Markdown ファイルリンクがビューア用 URL (`?content=xxx`) に自動変換されない仕様。
- 解消方法: `markdown.js` のリンク変換処理で、URLスキーム/拡張子に応じたルーティング処理を追加。
- 再発防止策: 今後ファイル分割やリンク遷移機能を拡張する際は、ブラウザ上での遷移先スキーマに考慮したパス解決ロジックを共通化する。
