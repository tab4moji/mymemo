## 2. 試行錯誤と決定事項
* **原因の調査**:
  * 既存のコードでは、すべての `a` タグに対して一律で `target="_blank"` および `rel="noopener noreferrer"` が設定されていた。
  * このため、同一ファイル内へのアンカーリンク（`#something`）に対しても新しいタブで開く挙動になっており、スクロール遷移が機能しなくなっていた。
  * また、 `.md` や `.markdown` で終わるローカルの相対ファイルリンク（例: `aipc.md`, `../README.md`）をクリックした際、ビューア（`markdown.html?content=xxx`）を介さずに生ファイルに直接ジャンプしようとしてしまうため、ブラウザ側で生テキストが表示されるかエラーになり、正常に閲覧できなかった。
  * **さらに追記された不具合**: ローカルの md 相対リンクを `markdown.html?content=xxx` に変換したとしても、同一ページ（`markdown.html`）でのクエリパラメータ変更遷移は、ブラウザによってはリロードが発生せず画面が更新されないため、「何も起きない（飛べない）」状態になってしまう問題が発覚した。
* **解決策の策定**:
  * `markdown.js` の `loadMarkdown` 内で `a` タグの属性設定をカスタマイズ。
    * **アンカーリンク (`#` で始まる)**: `target` 属性などを設定せず、そのままブラウザ標準のスクロール挙動に任せる。
    * **外部リンク (`http://`, `https://` で始まる)**: 従来通り安全に別タブで開く（`target="_blank"`, `rel="noopener noreferrer"`）。
    * **ローカルマークダウンファイル (`.md` や `.markdown` で終わる)**: 現在の表示ファイル名（`filename`）からの相対パスを `URL` API を用いて正しく解決し、 `markdown.html?content=解決したパス（拡張子なし）` の形式に `href` を書き換え、同じタブ（`target` なし）で開く。
    * **その他のローカルファイル (PDF等)**: 安全に別タブで開く。
  * **同一ページ遷移のSPA化**:
    * `contentDiv` でリンクのクリックイベントをインターセプト。
    * `markdown.html?content=` への遷移であれば `e.preventDefault()` で通常遷移を阻止し、 `history.pushState` で URL を書き換えた上で `loadMarkdown()` を直接呼ぶことで、ページ全体のリロードを挟まずに高速でドキュメントを切り替えるようにした。
    * 修飾キー（Ctrl, Cmd, Shift, Alt）が押されている場合は、ブラウザ標準の別タブオープン挙動を邪魔しないようにガードを実装。
    * 履歴の戻る・進むに対応するため `popstate` イベントも監視。
    * 非同期でマークダウンをロードした後に、ハッシュ（`#anchor`）がある場合は該当要素まで自動でスムーズスクロール（`scrollIntoView`）するよう実装。
* **決定事項**:
  * `URL` API (`new URL(baseDir + pathPart, 'http://dummy/')`) を用いることで、ディレクトリを遡る相対パス（`../` 等）やハッシュ付きの相対パス（`aipc.md#setup`）も簡潔に解決可能とした。
  * SPAライクなルーティングにすることで、動作の軽量化と「ページが遷移しない」問題の根本解決を両立した。

## 3. エラー調査結果
- 原因:
  1. `a` タグの一律 `target="_blank"` 適用。
  2. ローカル Markdown ファイルリンクがビューア用 URL (`?content=xxx`) に自動変換されない仕様。
  3. クエリのみが変化する同一ページへの遷移時にブラウザがリロードを行わない挙動。
- 解消方法: `markdown.js` にて URL 判別・変換を行い、さらに `pushState` / `popstate` / `loadMarkdown` の再実行およびロード後のハッシュ自動スクロールで解決した。
- 再発防止策: 今後ファイル分割やリンク遷移機能を拡張する際は、ブラウザ上での遷移先スキーマおよび履歴制御に考慮したパス解決・描画ロジックを設計する。
