# 2026-0625-0843-55: 見出しアンカー非表示不具合の解消と marked.js 互換性対応

## 2. 試行錯誤と決定事項
* 見出しアンカーが表示されない、または機能しない不具合の調査と解消。
* 原因1: 外部ライブラリ `marked.js` のバージョン差異（最新版のオブジェクト引数と、レガシーな個別引数の違い）により、レンダラーメソッド `heading` / `code` が正しく起動しない、または引数不足で動作しなかった可能性。対策として、引数オブジェクト形式と通常引数形式の双方に同時対応する防衛設計を `markdown_parse.js` に実装。
* 原因2: サニタイザー `DOMPurify` がデフォルト設定でカスタムデータ属性（`data-anchor`）を除外していた可能性。対策として、サニタイズ呼び出し時に `{ ALLOW_DATA_ATTR: true }` を付加。
* 原因3: ブラウザのキャッシュにより古いスタイルシートが保持され、テーマ変更やアンカー追加の CSS が反映されていなかった可能性。対策として、`markdown.html` の読み込みパスおよび `markdown.js` での動的テーマ切り替え用 `href` の双方にキャッシュバスター（`?v=2` およびタイムスタンプクエリ）を付与。

## 3. エラー調査結果 ※エラー関連の場合のみ記述
- 原因: `marked.js` バージョン差異による引数の不一致、および `DOMPurify` によるデータ属性の自動削除、CSSキャッシュ。
- 解消方法: レンダラーの引数両対応、サニタイズでの `ALLOW_DATA_ATTR: true` オプションの追加、および HTML / JS でのキャッシュバスターの設置。
- 再発防止策: 外部ライブラリとのインターフェース部では引数の型や構成を検証し、防衛的に処理するコードパターンを徹底する。
