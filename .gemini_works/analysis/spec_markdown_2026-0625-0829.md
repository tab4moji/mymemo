# 仕様・設計ドキュメント化: markdown_viewer (spec_markdown_2026-0625-0829.md)

⚠️ このドキュメントはリバースエンジニアリングおよびリファクタリング（ファイル分割）の成果として自動生成された。

## 概要
`markdown.html` から CSS、JS、テーマ定義を細かく分割し、保守性とカラーテーマの変更容易性を高めた高機能な Markdown ビューワーの設計仕様。

## 入出力・依存関係の定義

### 1. 入力 (Inputs)
- **URL パラメータ:** `?content=<filename>` を指定することで、読み込む Markdown ファイル名を決定。デフォルト値は `README`。
- **外部ファイル:** `${filename}.md`（キャッシュを回避するため、タイムスタンプクエリ `?t=<timestamp>` を付加して Fetch）。
- **ユーザー操作:** 
  - テーマ切り替えボタンのクリック
  - コードブロックの「Copy」ボタンクリック
  - 「Expand all」プレースホルダーのクリック

### 2. 出力 (Outputs)
- **サニタイズされた HTML:** `DOMPurify` でサニタイズされた上で、`#content` 要素にレンダリングされる。
- **ブラウザタイトル:** `<title>` を読み込まれたファイル名に動的に更新。
- **クリップボード:** 「Copy」ボタンクリック時にコードブロックのテキストを出力。

### 3. 副作用 (Side Effects)
- `localStorage` に `markdown-theme` のキー名で選択されたテーマ（`light` または `dark`）を保存。
- リンク (`<a>` タグ) に `target="_blank"` および `rel="noopener noreferrer"` 属性を付与。

### 4. 依存関係 (Dependencies)
- **外部 CDN スクリプト:**
  - `marked.min.js` (Markdown パース)
  - `purify.min.js` (HTML サニタイズ)
  - `highlight.min.js` & `powershell.min.js` (構文ハイライト)
- **ローカルリソース:**
  - `markdown.css` (共通レイアウトスタイル)
  - `markdown_dark.css` (ダークテーマ定義)
  - `markdown_light.css` (ライトテーマ定義)
  - `markdown_parse.js` (Markdown 修復・コンパイル処理)
  - `markdown.js` (アプリ動作・UI 制御)

## 機能詳細と実装ロジック

### <core_logic> 1. ネストされたコードブロック修復 (`fixNestedCodeBlocks`) </core_logic>
- **事実:** Markdown 内で多重ネストされたバッククォートの階層を解析し、外側のフェンス記号の長さが内側よりも長くなるよう自動調整し、パース崩れを防ぐ。
- **例外パス:** 入力テキストが空または文字列以外の場合、何もしないでそのまま値を返す。

### <core_logic> 2. テーマ切り替え機能 </core_logic>
- **事実:** OS設定 (`prefers-color-scheme`)、URLパラメータ `?theme=`、`localStorage` の優先順位で初期テーマをロード。
- **事実:** `theme-style` の ID を持つスタイルシート要素の `href` を `markdown_light.css` または `markdown_dark.css` に切り替える。
- **推論:** ユーザーの好むテーマ（ダーク・ライト）を保持し、次回アクセス時にシームレスに適用するための措置。

### <core_logic> 3. コードブロック折りたたみ (Snip) </core_logic>
- **事実:** コードブロックの行数が 20 行（`FOLD_THRESHOLD`）を超える場合、冒頭 10 行と末尾 10 行（`SHOW_LINES`）のみを表示し、中間を折りたたむ。
- **事実:** 折りたたみ領域をクリックすると、`hljs.highlightElement` が再適用され、コード全体が全展開されて表示される。

## 例外パスとエラーハンドリング

- **Fetch 失敗時 (404 またはネットワークエラー):**
  - `<article id="content">` 要素を非表示 (`display: none`) にする。
  - `<div id="error-message">` 要素を表示状態にし、エラーの原因メッセージ（例: `Error: Failed to load target_file`）をテキスト挿入する。
- **サニタイズ処理中の不正コード:**
  - `DOMPurify` が HTML パースの直後に無効なタグや悪意あるスクリプトを自動的に排除する。

## <constraints> 設計上の制約事項 </constraints>
- **ライブラリ依存:** クライアントサイドでの動作には、`marked.js`、`DOMPurify`、`highlight.js` の読み込みが前提となる。ローカルテストやオフライン環境では、これらのCDNへの接続が必要となる。
- **CSS変数:** カラーテーマは CSS カスタムプロパティ (Variables) を介して制御されているため、Internet Explorer などのレガシーブラウザはサポート対象外となる。
